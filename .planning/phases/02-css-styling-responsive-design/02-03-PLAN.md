<objective>
Implement mobile-first responsive design with proper breakpoints for mobile (1 col), tablet (2 col), and desktop (2-3 col) gallery layouts. Ensure all text is readable without pinch-to-zoom and touch targets are minimum 44px.
</objective>

<tasks>

<task>
**Task 3.1: Implement Mobile-First Responsive Breakpoints (20 minutes)**

Convert the current max-width media query to mobile-first min-width approach with proper breakpoints at 768px (tablet) and 1024px (desktop).

**Specific Actions:**
1. Remove the existing `@media (max-width: 720px)` block at the bottom of the file

2. Add tablet breakpoint `@media (min-width: 768px)`:
   ```css
   @media (min-width: 768px) {
     .gallery-grid {
       grid-template-columns: repeat(2, 1fr);
       gap: var(--space-lg);
       padding: var(--space-xl);
     }
     
     .site-nav {
       flex-direction: row;
       align-items: center;
     }
     
     body {
       font-size: 1.0625rem;
     }
   }
   ```

3. Add desktop breakpoint `@media (min-width: 1024px)`:
   ```css
   @media (min-width: 1024px) {
     .gallery-grid {
       grid-template-columns: repeat(3, 1fr);
       gap: var(--space-xl);
       padding: var(--space-2xl);
     }
     
     .featured-artwork {
       grid-column: 1 / 2;
       grid-row: 1 / 3;
       aspect-ratio: 2 / 3;
     }
     
     section {
       padding: var(--space-2xl) 0;
     }
   }
   ```

4. Verify mobile-first approach:
   - Default styles (no media query) apply to mobile
   - Tablet styles enhance for 768px+
   - Desktop styles enhance for 1024px+
   - No max-width queries used

5. Test at breakpoints:
   - Mobile: 375px, 414px
   - Tablet: 768px, 1024px
   - Desktop: 1280px, 1920px

**Success Criteria:**
- Gallery is 1 column on mobile
- Gallery is 2 columns on tablet (768px+)
- Gallery is 3 columns on desktop (1024px+)
- Featured image spans 2 rows on desktop
- Spacing increases at each breakpoint
- No layout breaks between breakpoints
</task>

<task>
**Task 3.2: Ensure Text Readability & Touch Targets (15 minutes)**

Verify all text is readable without pinch-to-zoom and all interactive elements have minimum 44px touch targets.

**Specific Actions:**
1. Verify font sizes on mobile:
   - Body text: `1rem` (16px) ✓
   - h1: `clamp(2.5rem, 4vw, 3.5rem)` (minimum ~40px on mobile) ✓
   - h2: `clamp(1.75rem, 2.5vw, 2.25rem)` (minimum ~28px on mobile) ✓
   - h3: `1.125rem` (18px) ✓
   - All text should be readable without zoom

2. Update form inputs to have minimum 44px height:
   ```css
   input,
   textarea {
     min-height: 44px;
     padding: var(--space-sm);
   }
   ```

3. Update button to ensure 44px minimum:
   ```css
   button {
     min-height: 44px;
     min-width: 44px;
     padding: var(--space-sm) var(--space-md);
   }
   ```

4. Update navigation links to have adequate touch target:
   ```css
   .site-nav a {
     padding: var(--space-xs) var(--space-sm);
     display: inline-block;
     min-height: 44px;
     display: flex;
     align-items: center;
   }
   ```

5. Test on mobile device:
   - All buttons are easily tappable
   - Form inputs are easy to interact with
   - Navigation links are not too close together
   - No accidental taps on wrong elements

**Success Criteria:**
- All text readable at 16px minimum on mobile
- No pinch-to-zoom required
- All buttons/inputs have 44px minimum height
- All interactive elements have adequate spacing
- Touch targets don't overlap
</task>

   ```css
     }
   }
   ```

   ```css
   main {
     padding: 0 var(--space-sm);  /* 16px on mobile */
   }
   
   @media (min-width: 768px) {

     }
   }
   ```

3. Update gallery grid padding:
   ```css
   .gallery-grid {
     padding: var(--space-lg);  /* 32px on mobile */
   }
   
   @media (min-width: 768px) {
     .gallery-grid {

   }
   
   @media (min-width: 1024px) {
     .gallery-grid {

     }
   ```


- Layout remains balanced at all sizes
