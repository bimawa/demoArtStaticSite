<objective>
Establish CSS foundation with design tokens (colors, spacing, typography variables), minimal reset, and Google Fonts integration. This creates the visual design system that all other styling will reference.
</objective>

<tasks>

<task>
**Task 1.1: Create CSS Variables & Design Tokens (15 minutes)**

Replace the current hardcoded values in styles.css with a comprehensive CSS variables system at the top of the file.

**Specific Actions:**
1. Add `:root` block with CSS variables for:
   - Colors: `--color-bg: #FDFBF7`, `--color-text: #1A1A1A`, `--color-border: #E8E6E1` (derived from cream palette)
   - Spacing: `--space-xs: 8px`, `--space-sm: 16px`, `--space-md: 24px`, `--space-lg: 32px`, `--space-xl: 48px`, `--space-2xl: 64px`
   - Typography: `--font-serif: 'Lora', Georgia, serif`, `--font-sans: 'Nunito', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
   - Transitions: `--transition-fast: 0.2s ease`

2. Update all existing hardcoded values to use variables:
   - Replace `#fafafa` with `var(--color-bg)`
   - Replace `#1f1f1f` with `var(--color-text)`
   - Replace `#dddddd` with `var(--color-border)`
   - Replace all padding/margin pixel values with appropriate spacing variables

3. Verify no hardcoded colors remain except in comments

**Success Criteria:**
- `:root` block is first in styles.css
- All color values use CSS variables
- All spacing values use CSS variables
- File still renders identically (no visual changes)
- Variables follow naming convention: `--category-name`
</task>

<task>
**Task 1.2: Add Google Fonts & Update Typography (15 minutes)**

Add Google Fonts import and update font-family declarations to use Lora and Nunito.

**Specific Actions:**
1. Add to the very top of styles.css (before `:root`):
   ```css
   @import url('https://fonts.googleapis.com/css2?family=Lora:wght@400;600&family=Nunito:wght@400;600;700&display=swap');
   ```

2. Update `body` font-family to: `font-family: var(--font-sans);`

3. Add new rule for headings (h1, h2, h3):
   ```css
   h1, h2, h3 {
     font-family: var(--font-serif);
     font-weight: 600;
   }
   ```

4. Update existing h1, h2, h3 rules to remove font-family declarations (they'll inherit from the new rule)

5. Verify fonts load by checking browser DevTools Network tab (should see googleapis.com requests)

**Success Criteria:**
- Google Fonts import is at the top of file
- Headings use Lora serif font
- Body text uses Nunito sans-serif font
- Fonts display correctly in browser (serif headings, sans-serif body)
- No FOIT (flash of invisible text) - fonts swap smoothly
</task>

<task>
**Task 1.3: Update Color Palette to Match Design System (15 minutes)**

Replace the current neutral grays with the warm cream and soft black palette from the design decisions.

**Specific Actions:**
1. Update `:root` color variables to exact values:
   - `--color-bg: #FDFBF7` (warm cream background)
   - `--color-text: #1A1A1A` (soft black text)
   - `--color-border: #E8E6E1` (light border, derived from cream)
   - `--color-border-dark: #D4D0C8` (darker border for more contrast if needed)

2. Update all color references throughout the file:
   - Body background: use `var(--color-bg)`
   - Body text color: use `var(--color-text)`
   - All borders: use `var(--color-border)`
   - Links and interactive elements: use `var(--color-text)`

3. Update focus outline color to match soft black: `outline: 2px solid var(--color-text);`

4. Verify contrast ratio: soft black (#1A1A1A) on warm cream (#FDFBF7) should be ~18:1 (exceeds WCAG AA requirement of 4.5:1)

**Success Criteria:**
- Background is warm cream (#FDFBF7)
- All text is soft black (#1A1A1A)
- No hardcoded color values remain (except in comments)
- Page has warm, minimal aesthetic
- Contrast is sufficient for readability
</task>
