<name>Task 1.1: Update gallery images to use picture element (20 minutes)</name>

<files>index.html</files>

<action>
Update all 4 artwork images in the gallery to use `<picture>` element with WebP source and JPEG fallback. This ensures browsers load the smallest supported format.

**Specific Actions:**

1. For each artwork `<figure>`, wrap the `<img>` in a `<picture>` element:

**Featured artwork (Tidal Memory):**
```html
<figure class="artwork featured-artwork">
  <picture>
    <source srcset="YOUR_WEBP_URL_HERE" type="image/webp">
    <source srcset="YOUR_JPEG_URL_HERE" type="image/jpeg">
    <img
      src="YOUR_JPEG_URL_HERE"
      alt="Tidal Memory, 2024, mixed media on canvas"
      width="1200"
      height="800"
      loading="eager"
    />
  </picture>
  <figcaption>
    <h3>Tidal Memory</h3>
    <p>2024</p>
    <p>Mixed media on canvas</p>
  </figcaption>
</figure>
```

2. Repeat for remaining 3 artworks (Quiet Geometry, Fragments of Light, Still Current) with their respective dimensions:
   - Quiet Geometry: 800x600
   - Fragments of Light: 800x600  
   - Still Current: 800x600

3. Replace placeholder URLs ("YOUR_WEBP_URL_HERE", "YOUR_JPEG_URL_HERE") with actual external URLs the user will provide. Use comment placeholders: `<!-- Replace with your WebP image URL -->` and `<!-- Replace with your JPEG image URL -->`

4. Keep existing width/height attributes (already present from Phase 1)

5. Set loading="eager" for the featured artwork (first image), keep loading="lazy" for others

**Important Notes:**
- The `<img>` tag inside `<picture>` should have the fallback URL as its `src`
- Width/height attributes MUST remain on the `<img>` element (prevents layout shift)
- `alt` text remains unchanged
- `loading` attribute goes on `<img>`, not `<picture>`
</action>
